# Contributing

Thanks for your interest in contributing to Canlab. We welcome bug reports, feature requests, documentaion improvements, and code contributions.

Please note that this porject follows a [Code of Conduct](CODE_OF_CONDUCT.md#code-of-conduct).
By contributing you agree to follow it.

## Getting Started
If you are new, look for issues labeled 'good first issue' and feel free to ask questions in issues or discussions.

## Reporting Bugs
To report bugs, open a new issue, and make sure to label the issue type as a bug.

When reporting a bug, please include:
- A clear description
- Steps to reproduce
- Expected and actual behavior
- Environmental details (OS, version, etc.)
- If possible code snippets or screenshots

## Suggusting Features
Feature requests are encouraged! We are trying to create a comprehensive CAN library so any feature requests are greatly appreciated.

Please describe:
- The problem you are trying to solve
- A proposed solution (if applicable)
- Other alternatives (if applicable)

## Development Setup
To contribute by developing follow these steps:
1. Fork the repo
2. Clone your fork
3. Install dependencies by running `uv sync`
4. Run tests to verify it has been setup properly

## Making Changes
- Create a feature branch from `main`
- Name it something meaningful such as the name of the feature you are implementing
- Keep commits focused and descriptive
- Follow existing code style

## Submiting a Pull Request
Before submitting a PR:
- Ensure all previous tests pass
- Write comprehensive tests for the new features if applicable
- Update documentation if needed
- Reference related issues

All PRs are reviewed before merging.
